# Deliverable templates

Use these as concise starting structures; replace brackets with evidence.

## README: Built with Codex

```md
## Built with Codex and GPT-5.6

**Product role of GPT-5.6.** [Describe the user-facing capability it powers and its input/output boundary.]

**How we collaborated with Codex.** Codex accelerated [planning / implementation / debugging / tests / design system / documentation] by [specific examples]. We retained responsibility for [key human product, engineering, and design decisions], including [examples].

**Evidence.** Core work was built in Codex session `[SESSION ID]`; relevant hackathon-period commits are [links]. For a pre-existing project, [HACKATHON_CHANGES.md](HACKATHON_CHANGES.md) distinguishes new work.

**Reliability and limits.** [Validation, fallbacks, known limitations, privacy/security notes, and test coverage.]
```

# [Product name]
**Tagline:** [specific outcome] for [specific audience].

## The problem
[Persona] loses [time/money/opportunity] when [recurring scenario]. Existing [workaround] fails because [specific gap].

## What it does
[Product] lets a user [primary flow] and receives [observable outcome]. GPT-5.6 is essential because [mechanism].

## Why this is different
[Defensible insight/mechanism], rather than [obvious alternative].

## Built with Codex and GPT-5.6
[Short, honest concrete evidence of both roles.]

## How to test
[URL/install command, account, sample input, expected output, reset instructions.]
```

## Demo beats (maximum 3:00)

| Time | Beat | Proof to show |
|---|---|---|
| 0:00–0:20 | Pain and audience | One concrete scenario; no broad claims. |
| 0:20–0:40 | Outcome and differentiation | The “aha” in a sentence. |
| 0:40–2:10 | Uninterrupted real workflow | Input → model/agent work → user decision/action → result. |
| 2:10–2:40 | Technology proof | Explain GPT-5.6 product role, Codex collaboration, and a reliability guardrail. |
| 2:40–3:00 | Impact and close | Measurable value, test URL, and memorable final frame. |
```

Avoid long intro animations, code tours, and scripted output presented as live behavior. Capture the real critical path first, then add voiceover.
