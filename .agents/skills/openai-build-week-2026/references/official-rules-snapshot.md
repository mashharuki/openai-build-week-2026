## Project and track requirements

- Build a project with **Codex and GPT-5.6** in exactly one best-aligned track: **Apps for Your Life**, **Work and Productivity**, **Developer Tools**, or **Education**.
- Accepted forms include web/native apps, backend tools, agent plugins (skills, MCPs, tools), and games.
- The project must install/run consistently on its intended platform and function as depicted in its materials.
- New projects must be made during the submission period. Pre-existing projects must be meaningfully extended with Codex and/or GPT-5.6 during that period; only the new work is evaluated. Document the distinction with dated commits, timestamped Codex sessions, or equivalent.
- Third-party SDKs, APIs, and data require authorization and compliance with their licences/terms.

## Required submission materials

- Select the most appropriate track and submit a feature/functionality text description.
- Provide a public YouTube demo video, **under 3 minutes**, with audio. It must clearly demo the product and explain use of Codex and GPT-5.6. Do not use third-party trademarks, copyrighted music, or other material without permission.
- Provide code repository URL: public with relevant licence, or private shared with **testing@devpost.com** and **build-week-event@openai.com**.
- README must explain collaboration with Codex, including where it accelerated the workflow, human product/engineering/design decisions, and how GPT-5.6/Codex contributed to the result.
- Provide the `/feedback` Codex Session ID for the project thread where most core functionality was built.
- Plugins/dev tools additionally need install instructions, supported platforms, and a way to test without rebuilding (demo instance, sandbox, or test account).
- Provide a working website/demo/test build. Private projects must include credentials. It must be free and unrestricted for judges through judging. Judges may judge only the description, images, and video rather than test it.
- Materials must be English or accompanied by English translations.
- A team/organization needs an eligible authorized representative.

## Eligibility and ownership

- Entrants must be adults where they live (with specified guardian provisions for students), and in OpenAI API-supported countries/territories, subject to listed exclusions and conflicts.
- Work must be original, owned by the entrant/team/organization, and not infringe IP, privacy, contract, trademark, or other rights.
- Open source is permitted when its licence is complied with and the entrant adds meaningful enhancement.
- Multiple entries are allowed only if each is unique and substantially different.
- A submission cannot have been developed/derived with financial or preferential support from the sponsor/administrator before the submission period ends.

## Judging

Stage 1 is a viability pass/fail: the project must reasonably fit the theme and reasonably apply the required featured APIs/SDKs.

Stage 2 scores four **equally weighted** criteria:

1. **Technological Implementation** — thorough, skillful Codex use; genuine, working, non-trivial code.
2. **Design** — complete, coherent, working/runnable product experience, not only a technical proof of concept.
3. **Potential Impact** — credible, specific real problem/audience and a demonstrated fit between problem and solution.
4. **Quality of the Idea** — creative, novel concept differentiated from existing concepts.

Ties are first broken by technological implementation, then the remaining criteria in order.

## Prizes

Each track has one $15,000 first-place and one $10,000 second-place prize (eight cash prizes total; $100,000). First place also includes up to two DevDay/Exchange passes, OpenAI Developers promotion, a Codex team meeting, and one year of Pro; verify all prize details in the live rules. A project is eligible for one prize.
